# example-app — Task API

The "horse." A deliberately small project that an agent develops **under the harness**
we build in `../harness/`. Kept minimal on purpose: the lessons are about the harness,
not the app.

Planned surface (filled in around L8 when we write the feature list):
- `POST /tasks`, `GET /tasks`, `GET /tasks/:id`, `PATCH /tasks/:id`, `DELETE /tasks/:id`

Nothing is implemented yet — we add features only once the harness can define,
verify, and track them.
